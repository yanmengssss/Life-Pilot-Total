# 智能生活管家系统的设计与实现

# 第1章 引言

## 1.1 研究背景与研究意义

在信息碎片化时代，人们每天要处理大量的日程、任务和待办事项。传统待办事项管理工具大多依赖用户手动录入，缺乏智能化的辅助手段来帮助用户规划时间、分解任务。这种纯手动管理的方式效率低下，对于需要同时兼顾学习、工作和生活多个维度的用户群体而言，繁琐的录入操作反而成为了额外的负担。

随着大语言模型（LLM）技术的快速发展，特别是以GPT-4为代表的多模态大模型的出现，AI在自然语言理解、上下文推理和任务规划方面的能力有了质的飞跃。将LLM与待办事项管理相结合，打造一个能够理解自然语言指令、自动分解复杂任务并主动规划时间的智能助手，成为了一个值得探索的方向。

与此同时，WebRTC技术的成熟使得浏览器端实时音视频通信成为可能，为语音交互提供了基础设施保障。LangGraph等Agent工作流框架的出现，则为构建复杂的多步骤AI任务规划提供了可靠的技术支撑。这些技术的交叉融合，使得开发一个以AI为核心的智能日程与任务助理成为现实。本项目LifePilot正是基于这样的技术背景应运而生。

LifePilot项目的研发具有以下几方面的研究意义和实践价值：

（1）探索LLM在实际生产环境中的应用落地。本项目将LangGraph工作流与MCP协议相结合，验证了如何通过标准化的协议接口让AI Agent安全、可控地访问数据库，完成实际的任务CRUD操作。这对于类似的AI Agent应用开发具有参考价值。

（2）验证前后端分离架构下AI对话系统的工程实现方案。本项目采用了Next.js前端+Express后端的经典分离架构，通过SSE实现流式输出，通过MCP协议对接外部工具服务。这些实践为同类系统的开发提供了可借鉴的工程经验。

（3）整合多种外部API能力，构建完整的RAG知识库系统。本项目整合了七牛云OSS、SiliconFlow语音识别、高德地图MCP和小红书MCP等多种外部服务，并通过向量数据库Weaviate实现了私有文档的语义检索能力。

（4）形成可复用的前端组件库和SDK。本项目开发了yanmengs-logs和yanmengs-rag-package两个npm包，实现了日志上报和AI能力调用的SDK化封装，提高了代码复用性。

## 1.2 研究内容与论文结构安排

本论文主要研究内容包括以下几个方面：

（1）智能任务规划系统的设计与实现。研究如何基于LangGraph构建多节点AI工作流，实现自然语言任务解析、自动分解、计划生成和用户确认的完整流程。重点解决如何通过三角色模式保证AI输出质量，并通过人在回路机制确保用户对AI生成计划的有效控制。

（2）基于MCP协议的工具服务架构。研究如何通过Model Context Protocol构建标准的工具服务层，实现AI Agent与后端数据库的安全隔离。

（3）多模态交互能力的研究。包括语音识别（ASR）、文本转语音（TTS）的流式播放、以及文档上传后的RAG检索问答。

（4）系统架构与部署方案的研究。研究如何通过Docker容器化和Jenkins CI/CD实现多个微服务的自动化构建与部署。

本论文共分为七章，各章内容安排如下：

第1章 引言，阐述研究背景、研究意义、主要研究内容和论文结构。

第2章 相关技术介绍，介绍本项目所使用的核心技术栈，包括Next.js前端框架、LangGraph工作流、MCP协议、WebRTC音视频通信、RAG向量检索等技术的原理与应用。

第3章 系统需求分析，对LifePilot系统进行需求分析，包括功能需求分析、用例分析和非功能需求分析。

第4章 系统设计，阐述系统的总体架构设计、各子模块的详细设计、数据库设计，以及核心工作流的详细设计。

第5章 系统实现，描述主要功能模块的具体实现过程，包括前端页面实现、后端API实现、AI对话流程实现等。

第6章 系统测试，对系统进行功能测试和性能测试，验证系统的正确性和稳定性。

第7章 总结与展望，总结本论文的主要工作，展望未来的改进方向。
