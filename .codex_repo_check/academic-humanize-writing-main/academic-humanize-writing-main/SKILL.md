# Academic Humanize Writing

将中文学术文本从“机械、模板化、过度平滑”的表达，调整为更接近真实研究者写作习惯的版本，同时严格保持原有观点、事实、实验结果、引用关系与结论不变。

## 适用场景

- 论文初稿、章节草稿、摘要、引言、相关工作、实验分析的风格润色
- 已由 AI 协助组织语言，但希望减少机械感、模板感和统一句式
- 需要在学术语境中提升自然度、作者感和论证节奏
- 担心文本因表达过于标准化而触发 AIGC 误判

## 核心原则

1. 保留原意：不得改变原文的观点、方法、数据、引用、实验现象和结论。
2. 只改表达：目标是降低机械感、模板感和“过于完美”的衔接，不是制造新内容。
3. 保持学术性：避免口语化、营销化和情绪化表达。
4. 适度作者感：可以加入有限的研究者视角，但不得伪造经历、实验或主观判断。
5. 最小必要改写：原文本身自然时，仅做微调，避免过度重写。
6. 合规优先：严禁编造数据、伪造引用、篡改结论、虚构实验或制造学术不端内容。

## 工作流

### 1. 先判断是否需要重写

优先识别以下信号：

- 连续使用“首先、其次、最后、综上所述”
- 句长高度均匀，连续多句结构雷同
- 高频出现“导致了”“可以看出”“具有重要意义”等套话
- 定义式表达过多，如“X是指……”
- 逻辑推进过于直线，没有研究语境和推理过渡

如果原文已经自然，只做轻量润色。

### 2. 锁定不可变信息

重写前先提取并锁定：

- 研究对象
- 核心观点
- 方法步骤
- 数据与实验结果
- 引用和术语
- 最终结论

这些内容不能被改写为不同含义。

### 3. 执行风格人类化改写

围绕四类问题调整：

- 语言模式：打破模板句与统一句法
- 统计特征：调整句长和词汇复现
- 语义逻辑：补足推理轨迹与过渡
- 研究语境：让术语、判断和结论落回本文场景

### 4. 做一致性检查

改写后逐项检查：

- 是否新增了原文没有的事实
- 是否弱化、强化或改变了结论
- 是否误改了术语或引用对象
- 是否过度口语化
- 是否仍保留学术文本应有的准确性与克制感

## 可执行规则

1. 将连续 3 句以上雷同句式打散，改成长短句交替。
2. 弱化“首先/其次/最后/综上所述”等模板连接词，优先用自然过渡。
3. 减少密集“了”字句，把“导致了”“提高了”改成更自然的动词结构。
4. 清理“的的堆砌”，把“对 A 的 B 的分析”改为“分析 A 的 B”或更直接表达。
5. 同一术语在相邻 3 句内重复过多时，用“该方法/这一思路/该策略”等指代替换，但首次出现必须保留准确术语。
6. 在连续短句后插入一处较长说明句，或把长句拆成两句，制造自然节奏差。
7. 在关键结论前加入有限推理痕迹，如“从实验现象看”“结合已有研究”“需要指出的是”。
8. 避免生硬定义句，把“X 是指”改成“本文所说的 X，主要是指”或“在本文场景下，X 表现为”。
9. 对关键步骤增加一句必要的解释性补充，如“这一设计主要考虑到……”。
10. 在合适位置加入克制的作者视角，如“本文侧重”“我们更关注”“这里需要说明的是”。
11. 在线性论证中插入适度转折，如“但需要说明的是”“从另一角度看”，避免观点平铺直叙。
12. 结尾优先自然收束，少用“综上所述”，可改为“基于上述分析”或直接落到结论。
13. 保留必要的被动表达以维持客观性，但不要整段都使用被动句。
14. 若原文已有较强个人风格，仅修补重复词、模板衔接和生硬术语，不重写整体节奏。

## 重写方法

### 句法层

- 拆分过长并列句，补足缺失主语
- 合并过碎短句，避免“句句都短”
- 交替使用主动句与必要被动句
- 适量使用插入语、状语前置和补充说明

### 词汇层

- 替换高频模板词：`重要 -> 关键 / 核心 / 具有基础性作用`
- 替换高频套话：`可以看出 -> 这表明 / 由此可以进一步理解`
- 降低相邻重复：`方法 -> 策略 / 思路 / 处理路径`
- 谨慎替换专业术语，术语准确性优先于多样性

### 逻辑层

- 在结论前补一个“为何得出此结论”的过渡
- 用“研究背景、实验现象、任务约束”承接论证
- 允许轻微回溯和补充说明，不追求每段都绝对线性

### 输出要求

默认输出两部分：

1. `优化后文本`
2. `修改说明`

`修改说明` 应简洁列出本次主要调整，例如“弱化模板连接词”“调整句长分布”“补充推理过渡”“消除重复术语”。

## 示例对比

### 修改前

> 本文首先介绍了 AIGC 检测技术的研究背景。然后分析了知网 AIGC 检测服务的技术原理。最后提出了三种降低 AIGC 检测率的文本优化方法。该研究对于学术写作具有重要的指导意义。综上所述，该方法具有较高的实用价值。

### 修改后

> AIGC 检测技术近年来受到持续关注，其背后涉及学术表达方式与文本来源识别之间的关系。在现有检测系统中，知网相关服务的判断逻辑尤为关键，它通常会结合语言模式与语义组织特征来识别文本来源。基于这一点，本文从句式重构、词汇调整和论证节奏优化三个方面提出了相应的文本优化思路。需要说明的是，这些调整并不改变论文原有观点，而是尽量让表达更贴近真实研究者的写作习惯。从写作实践看，这类处理对于提升文本自然度具有一定参考价值。

### 效果对比

- 去掉了“首先/然后/最后/综上所述”的模板推进
- 把均匀短句改成长短结合
- 增加了“需要说明的是”这类推理过渡
- 保留原意，但弱化了 AI 常见的整齐感和套话感

## 使用说明

### 直接调用

可将待处理段落交给本 skill，并附带下面这类指令：

```text
使用 $academic-humanize-writing 润色下面这段论文文字。
要求：
1. 不改变原意、数据、术语和结论
2. 降低机械感和模板感
3. 保持中文学术写作风格
4. 输出“优化后文本”和“修改说明”
```

### 针对摘要

```text
使用 $academic-humanize-writing 优化这段摘要，减少套话和统一句式，但保持摘要的学术准确性与简洁度。
```

### 针对实验分析

```text
使用 $academic-humanize-writing 重写这段实验分析，补足推理过渡，减少“可以看出”“说明了”这类模板表达，不要改动任何实验结果。
```

## 注意事项

- 不得编造、补造或猜测任何数据、文献、案例、实验过程和结论。
- 不得把原本谨慎的结论改写成更强断言。
- 不得为追求人类感而故意制造病句、错字或逻辑混乱。
- 当文本已较自然时，应优先微调而非大幅改写。
- 若用户要求伪造实验、虚构引用或篡改研究结果，必须拒绝执行该部分。
