# 运维手册

## 第一章 系统概述

本手册涵盖生产环境运维工作的规范流程、常见问题处理方法和应急响应机制。

## 第二章 日常运维

### 2.1 巡检流程

**每日巡检项目**
| 检查项 | 检查内容 | 频率 | 超时阈值 |
|--------|----------|------|----------|
| 服务器CPU | CPU使用率 | 每小时 | >80%持续5分钟 |
| 内存使用 | 内存占用率 | 每小时 | >85% |
| 磁盘空间 | 磁盘使用率 | 每4小时 | >80% |
| 数据库 | 连接数、慢查询 | 每小时 | >100慢查询 |
| 接口响应 | API平均响应时间 | 每5分钟 | >500ms |
| 错误率 | 业务接口错误率 | 每5分钟 | >1% |

### 2.2 日志管理

**日志级别定义**
- DEBUG：调试信息，生产环境关闭
- INFO：正常业务日志
- WARNING：警告信息，需要关注
- ERROR：错误信息，需要处理
- FATAL：致命错误，需立即处理

**日志保留策略**
- 应用日志：保留30天
- 访问日志：保留90天
- 错误日志：保留180天
- 数据库日志：保留7天

## 第三章 中间件运维

### 3.1 Nginx配置

**常用命令**
```bash
# 检查配置语法
nginx -t

# 平滑重载配置
nginx -s reload

# 优雅停止
nginx -s quit

# 查看运行状态
systemctl status nginx
```

**性能优化参数**
- worker_processes: auto
- worker_connections: 65535
- keepalive_timeout: 65
- client_max_body_size: 100m

### 3.2 MySQL运维

**常用维护命令**
```sql
-- 检查连接数
SHOW STATUS LIKE 'Threads_connected';

-- 查看慢查询
SHOW FULL PROCESSLIST;

-- 优化表
OPTIMIZE TABLE table_name;

-- 分析表
ANALYZE TABLE table_name;
```

**备份策略**
- 全量备份：每周日凌晨2点
- 增量备份：每天凌晨2点
- 备份保留：最近30天
- 备份验证：每周验证一次

### 3.3 Redis运维

**常用命令**
```bash
# 查看内存使用
info memory

# 查看key数量
dbsize

# 持久化保存
save / bgsave

# 监控实时操作
monitor
```

## 第四章 监控告警

### 4.1 告警等级

| 等级 | 名称 | 响应时间 | 处理方式 |
|------|------|----------|----------|
| P0 | 紧急 | 5分钟 | 电话+短信+立即处理 |
| P1 | 严重 | 15分钟 | 短信+立即处理 |
| P2 | 一般 | 1小时 | 邮件+工作日处理 |
| P3 | 提醒 | 4小时 | IM通知 |

### 4.2 告警规则

**服务器告警**
- CPU持续5分钟 >90%
- 内存使用 >85%
- 磁盘空间 <10%
- 服务进程退出

**应用告警**
- QPS下降 >50%
- 错误率 >5%
- 平均响应时间 >2秒
- 数据库连接池耗尽

## 第五章 故障处理

### 5.1 故障分级

| 级别 | 定义 | 示例 |
|------|------|------|
| 一级 | 全站不可用 | 首页无法访问 |
| 二级 | 核心功能不可用 | 下单功能异常 |
| 三级 | 非核心功能异常 | 消息推送延迟 |
| 四级 | 性能下降 | 响应时间增加 |

### 5.2 故障处理流程

1. 故障发现：通过监控或用户报告
2. 故障确认：判断影响范围和等级
3. 故障定位：查看日志、监控、链路追踪
4. 故障处理：执行降级、限流、回滚等操作
5. 故障恢复：验证服务恢复正常
6. 故障总结：复盘分析，制定改进措施

### 5.3 常用回滚操作

**应用回滚**
```bash
# 查看历史版本
kubectl rollout history deployment/app-name

# 回滚到上一版本
kubectl rollout undo deployment/app-name

# 回滚到指定版本
kubectl rollout undo deployment/app-name --to-revision=2
```
